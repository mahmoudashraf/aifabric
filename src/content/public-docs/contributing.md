# Contributing

AI Fabric is open source under Apache License 2.0. The best first contributions improve developer trust: tests, docs, examples, small bug fixes, and real app scenarios.

## Before Opening A Change

1. Read the [getting started guide](/docs/getting-started).
2. Identify the module boundary you are changing.
3. Add or update tests that preserve current behavior.
4. Run the focused module tests first, then the broader framework verification when the change crosses module boundaries.

## Good First Issue Areas

- Improve getting-started examples.
- Add small tests around public APIs.
- Improve real-app README files.
- Add demo reset or health-check clarity.
- Add provider-specific documentation.
- Improve error messages without changing behavior.
